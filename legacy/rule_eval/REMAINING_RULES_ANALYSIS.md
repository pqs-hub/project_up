#### T03 - Case重排序（15.63%）

**规则描述**：重排序case分支

**可能原因**：
```verilog
// 原始
case(sel)
  0: out = a;
  1: out = b;
  default: out = c;
endcase

// 重排后
case(sel)
  1: out = b;
  0: out = a;
  default: out = c;
endcase
```

**为什么会改变功能**：
1. **优先级问题**：如果case不是完全并行的（如有重叠条件）
2. **综合工具差异**：某些工具可能对顺序敏感
3. **default分支**：重排可能影响default的触发条件

**建议**：
- 15.63%的改变率相对较低
- 如果可以接受，可以使用
- 或者添加检查：只在没有default分支时重排

---

#### T09 - DeMorgan AND（20.40%）

**规则描述**：`a & b` → `~(~a | ~b)`

**可能原因**：
```verilog
// 原始
assign out = a & b;

// 变换后
assign out = ~(~a | ~b);
```

**为什么会改变功能**：
1. **位宽不匹配**：如果a和b位宽不同
2. **未定义值**：X/Z值的处理可能不同
3. **符号扩展**：负数的处理可能不同

**建议**：
- 20.40%的改变率需要关注
- 可以添加位宽检查：只在a和b位宽相同时应用
- 或者添加类型检查：只在无符号信号上应用

**建议**：

#### T48 - 反拓扑排序（25.06%）

**规则描述**：颠倒assign语句顺序

**可能原因**：
```verilog
// 原始

assign b = y;
assign c = a + b;

// 反转后
assign c = a + b;  // ❌ a和b还未定义！
assign b = y;

```

**为什么会改变功能**：
1. **数据依赖**：如果存在依赖关系，顺序很重要
2. **初始化顺序**：某些综合工具对初始化顺序敏感

**建议**：
- ⚠️ **需要添加依赖检查**
- 只在没有数据依赖时反转
- 或者使用拓扑排序确保依赖关系
