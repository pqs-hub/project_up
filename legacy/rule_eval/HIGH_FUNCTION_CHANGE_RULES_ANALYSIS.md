# 高功能改变率规则分析

## 问题

5个规则（T47, T38, T31, T11, T37）导致了较高的功能改变率（>40%），这可能不符合"功能等价变换"的预期。

## 规则分析

### T47 - Dataflow Shatter（Shannon展开）

**功能改变率**: 95.0%

**规则描述**: Shannon展开比较器/加法器

**实现**:
```python
'T47': Transform(
    'T47', 'Dataflow Shatter', 'Shannon expansion', 
    ast_dataflow_shattering, 'structure', 5,
    params=[
        ParamSpec(
            name='width',
            type='int',
            default=4,
            description='位宽（用于 Shannon 展开）',
            min_val=2,
            max_val=16
        )
    ]
)
```

**为什么会改变功能**:
- Shannon展开是一种**逻辑等价变换**，理论上应该功能等价
- 但95%的改变率说明**实现可能有bug**
- 可能的原因：
  1. 位宽处理不当
  2. 边界条件处理错误
  3. 展开逻辑有误

**建议**: 
- ⚠️ **需要修复此规则的实现**
- 检查Shannon展开的逻辑是否正确
- 验证位宽匹配

## 根本原因分析

### 1. 规则版本不匹配

当前数据集（`metrics_conf_v2_on_fullall_adv`）中包含的规则（T11, T37, T38）在当前的`ast_transforms.2.py`中找不到，说明：
- 这些是**旧版本的规则**
- 数据集是用旧版本的代码生成的
- 规则可能已被重构或删除

### 2. 实现Bug

对于仍在使用的规则（T47, T31）：
- 理论上应该是功能等价变换
- 但实际改变率很高
- 说明**实现有bug**

### 3. 测试不充分

- 规则在实现时可能没有充分测试
- 没有验证功能等价性
- 边界条件处理不当
