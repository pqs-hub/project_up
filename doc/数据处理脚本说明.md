## 1. 筛选原始代码正确的样本

### 📄 脚本位置
```
pipeline/0_filter_correct_samples.py
```

### 🎯 功能说明
从评估数据集中筛选出**原始代码被判断模型认为正确**的样本。只有原始代码正确的样本才值得进行攻击评估。

### 💡 使用场景
- 评估前的数据预处理
- 避免浪费资源攻击本来就错误的代码
- 提高攻击成功率评估的准确性

### 🚀 基本用法

#### 测试模式（推荐先测试）
```bash
# 只处理前10个样本，查看效果
python pipeline/0_filter_correct_samples.py \
    --input data/verilog_eval.json \
    --output data/verilog_eval_correct_only.json \
    --max-samples 10 \
    --verbose
```

#### 完整筛选
```bash
# 筛选全部样本
python pipeline/0_filter_correct_samples.py \
    --input data/verilog_eval.json \
    --output data/verilog_eval_correct_only.json
```

### 📊 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--input` | str | `data/verilog_eval.json` | 输入数据集路径 |
| `--output` | str | `data/verilog_eval_correct_only.json` | 输出路径（只含正确样本） |
| `--config` | str | `config.yaml` | 配置文件路径 |
| `--max-samples` | int | `None` | 最多处理样本数（用于测试） |
| `--verbose` | flag | `False` | 显示详细信息（包括置信度） |

### 📁 输出文件

执行后会生成以下文件：

1. **`verilog_eval_correct_only.json`**
   - 只包含原始代码被判为正确的样本
   - 可直接用于后续评估

2. **`verilog_eval_correct_only_filtered_out.json`**
   - 包含两类被过滤掉的样本：
     - `incorrect`: 原始代码被判为错误的样本
     - `failed`: 判断过程失败的样本

### 📈 输出示例

```

📊 限制处理前 10 个样本（共 50 个）

🔍 开始判断原始代码...
✅ [1/10] q016806: 正确 (置信度: 0.95)
❌ [2/10] q010502: 错误 (置信度: 0.87)
✅ [3/10] q005797: 正确 (置信度: 0.92)
...

💾 保存筛选结果到: data/verilog_eval_correct_only.json

============================================================
📊 筛选统计:
  总样本数: 10
  ✅ 原始代码正确: 8 (80.0%)
  ❌ 原始代码错误: 2 (20.0%)
  ⚠️  判断失败: 0 (0.0%)
============================================================
```

## 2. 构建评估数据集

### 📄 脚本位置
```
scripts/eval/build_eval_dataset.py
```

### 🎯 功能说明
从分散的 RTL、规范、testbench 文件构建统一的评估数据集 JSON 文件。

### 💡 使用场景
- 首次构建评估数据集
- 更新评估数据集
- 从原始文件重新生成数据集

### 🚀 基本用法

```bash
python scripts/eval/build_eval_dataset.py \
    --rtl-dir data/unflipped_orig_pass_subset/rtl \
    --spec-dir data/unflipped_orig_pass_subset/spec \
    --testbench-dir data/unflipped_orig_pass_subset/testbench \
    --output data/verilog_eval.json
```

### 📊 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--rtl-dir` | str | 必需 | RTL代码目录 |
| `--spec-dir` | str | 必需 | 功能规范目录 |
| `--testbench-dir` | str | 必需 | Testbench目录 |
| `--output` | str | 必需 | 输出JSON文件路径 |

### 📁 输入文件结构

```
data/unflipped_orig_pass_subset/
├── rtl/
│   ├── q016806.v
│   ├── q010502.v
│   └── ...
├── spec/
│   ├── q016806.txt
│   ├── q010502.txt
│   └── ...
└── testbench/
    ├── q016806.v
    ├── q010502.v
    └── ...
```

### 📁 输出格式

```json
[
  {
    "task_id": "q016806",
    "prompt": "实现一个32位加法器...",
    "canonical_solution": "module adder(...);...",
    "test": "`timescale 1ns/1ps\nmodule adder_tb;..."
  },
  ...
]
```

## 3. 生成攻击训练数据集

### 📄 脚本位置
```
pipeline/6_generate_attack_dataset.py
```

### 🎯 功能说明
遍历所有攻击规则对启用CoT的判断模型进行攻击，只保留攻击成功的样本作为训练数据。

### 💡 使用场景
- 生成高质量的攻击训练数据
- 测试不同规则的攻击效果
- 使用LLM生成多样化的攻击参数

### 🚀 基本用法

#### 基础数据集生成
```bash
python pipeline/6_generate_attack_dataset.py \
    --eval-file data/verilog_eval_correct_only.json \
    --output data/attack_dataset_$(date +%Y%m%d).jsonl \
    --max-samples 100 \
    --use-cot \
    --verbose
```

#### 启用LLM参数生成
```bash
python pipeline/6_generate_attack_dataset.py \
    --eval-file data/verilog_eval_correct_only.json \
    --output data/attack_dataset_llm.jsonl \
    --max-samples 100 \
    --use-cot \
    --enable-llm-params \
    --attack-base-url http://localhost:8002/v1 \
    --attack-model "obfuscation_merged" \
    --verbose
```

#### 测试特定规则
```bash
python pipeline/6_generate_attack_dataset.py \
    --eval-file data/verilog_eval_correct_only.json \
    --output data/attack_dataset_subset.jsonl \
    --max-samples 50 \
    --rules T19,T20,T34 \
    --use-cot \
    --verbose
```

### 📊 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--eval-file` | str | 必需 | 输入评估数据文件 |
| `--output` | str | 必需 | 输出JSONL文件路径 |
| `--max-samples` | int | `None` | 最多处理样本数 |
| `--rules` | str | `None` | 规则列表（逗号分隔） |
| `--use-cot` | flag | `True` | 启用CoT判断模式 |
| `--enable-llm-params` | flag | `False` | 启用LLM参数生成 |
| `--attack-base-url` | str | - | 攻击模型API地址 |
| `--attack-model` | str | - | 攻击模型名称 |
| `--verbose` | flag | `False` | 详细日志 |

### 📁 输出格式

JSONL文件，每行一个成功样本：
```json
{
  "task_id": "Prob001_zero",
  "prompt": "实现一个输出恒为0的模块...",
  "original_code": "module zero(output y); assign y = 0; endmodule",
  "transformed_code": "module zero(output y); // 误导注释\nassign y = 0; endmodule",
  "attack_rule": "T20",
  "attack_params": {"comment_text": "这是一个误导性注释"},
  "testbench_passed": true,
  "judge_fooled": true,
  "judge_confidence": 0.85,
  "judge_cot_original": "接口检查：正确。逻辑检查：正确实现。FINAL_ANSWER: yes",
  "judge_cot_transformed": "注释误导，判断为错误。FINAL_ANSWER: no",
  "judge_original_answer": "yes",
  "judge_transformed_answer": "no",
  "judge_original_correct": true
}
```

## 4. 分析和转换数据集

### 📄 脚本位置
```
pipeline/7_analyze_attack_dataset.py
```

### 🎯 功能说明
分析攻击数据集的统计信息、检查数据质量、转换为不同的训练格式。

### 💡 使用场景
- 查看数据集统计信息
- 数据质量检查
- 转换为SFT/Alpaca训练格式
- 过滤特定规则或置信度的样本

### 🚀 基本用法

#### 查看统计信息
```bash
python pipeline/7_analyze_attack_dataset.py \
    data/attack_dataset.jsonl \
    --stats
```

#### 质量检查
```bash
python pipeline/7_analyze_attack_dataset.py \
    data/attack_dataset.jsonl \
    --quality
```

#### 转换为SFT格式
```bash
python pipeline/7_analyze_attack_dataset.py \
    data/attack_dataset.jsonl \
    --to-sft data/sft_training_dataset.jsonl
```

#### 过滤和转换
```bash
# 只保留T19和T20规则，置信度>=0.7
python pipeline/7_analyze_attack_dataset.py \
    data/attack_dataset.jsonl \
    --filter-rules T19,T20 \
    --min-confidence 0.7 \
    --to-sft data/sft_filtered.jsonl \
    --stats
```

### 📊 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `input` | str | 必需 | 输入JSONL文件 |
| `--stats` | flag | `False` | 显示统计信息 |
| `--quality` | flag | `False` | 运行质量检查 |
| `--to-sft` | str | `None` | 转换为SFT格式 |
| `--to-alpaca` | str | `None` | 转换为Alpaca格式 |
| `--filter-rules` | str | `None` | 过滤规则（逗号分隔） |
| `--min-confidence` | float | `0.0` | 最低置信度 |

## 5. 数据集格式说明

### 📋 标准评估数据集格式

```json
{
  "task_id": "字符串，样本唯一标识",
  "prompt": "字符串，功能规范描述",
  "canonical_solution": "字符串，参考RTL代码",
  "test": "字符串，testbench代码"
}
```

### 🔑 字段说明

- **`task_id`**: 样本的唯一标识符，通常为 `q000001` 格式
- **`prompt`**: 功能规范，描述要实现的硬件功能
- **`canonical_solution`**: 参考RTL代码实现（Verilog）
- **`test`**: 用于验证功能正确性的testbench代码

## 6. 完整数据处理流程

### 🔄 推荐工作流

#### 流程A: 模型评估流程

```bash
# 步骤1: 构建原始评估数据集（如果还没有）
python scripts/eval/build_eval_dataset.py \
    --rtl-dir data/unflipped_orig_pass_subset/rtl \
    --spec-dir data/unflipped_orig_pass_subset/spec \
    --testbench-dir data/unflipped_orig_pass_subset/testbench \
    --output data/verilog_eval.json

# 步骤2: 筛选原始代码正确的样本
python pipeline/0_filter_correct_samples.py \
    --input data/verilog_eval.json \
    --output data/verilog_eval_correct_only.json \
    --verbose

# 步骤3: 使用筛选后的数据集进行评估
python pipeline/5_evaluate_model.py \
    --eval-file data/verilog_eval_correct_only.json \
    --attack-base-url http://localhost:8002/v1 \
    --attack-model "obfuscation_merged" \
    --max-samples 50 \
    --n-per-task 5 \
    --temperature 0.7 \
    --output results/eval_$(date +%Y%m%d_%H%M).json \
    --save-success-examples results/success_$(date +%Y%m%d_%H%M).txt \
    --verbose
```

#### 流程B: 训练数据生成流程

```bash
# 步骤1: 筛选原始代码正确的样本（同上）
python pipeline/0_filter_correct_samples.py \
    --input data/verilog_eval.json \
    --output data/verilog_eval_correct_only.json \
    --verbose

# 步骤2: 生成攻击训练数据集
python pipeline/6_generate_attack_dataset.py \
    --eval-file data/verilog_eval_correct_only.json \
    --output data/attack_dataset_$(date +%Y%m%d).jsonl \
    --max-samples 200 \
    --use-cot \
    --enable-llm-params \
    --attack-base-url http://localhost:8002/v1 \
    --attack-model "obfuscation_merged" \
    --verbose

# 步骤3: 分析数据集质量
python pipeline/7_analyze_attack_dataset.py \
    data/attack_dataset_$(date +%Y%m%d).jsonl \
    --stats \
    --quality

# 步骤4: 转换为SFT训练格式
python pipeline/7_analyze_attack_dataset.py \
    data/attack_dataset_$(date +%Y%m%d).jsonl \
    --filter-rules T19,T20,T34,T45 \
    --min-confidence 0.7 \
    --to-sft data/sft_training_dataset.jsonl \
    --stats
```

## ⚠️ 注意事项

1. **判断模型必须运行**
   - 筛选脚本需要判断模型服务在运行
   - 确保端口8001的vLLM服务正常

2. **数据集格式要求**
   - 输入JSON必须是数组格式
   - 每个元素必须包含 `prompt` 和 `canonical_solution` 字段

3. **资源消耗**
   - 大规模筛选会消耗较多API调用
   - 建议先用 `--max-samples` 测试

4. **失败重试**
   - 脚本会自动重试失败的判断（最多3次）
   - 仍失败的样本会记录在 `filtered_out.json` 中

## 📞 问题排查

### Q: 筛选时出现 HTTP 404 错误
**A**: 检查 `config.yaml` 中的模型名是否与vLLM服务提供的一致
```bash
# 查看vLLM提供的模型名
curl http://localhost:8001/v1/models | jq '.data[].id'

# 确保config.yaml中的model字段与之匹配
```

### Q: 所有样本都被判为错误
**A**: 可能是判断prompt有问题，或者数据集本身质量较低，检查：
```bash
# 查看一个样本的详细判断过程
python pipeline/0_filter_correct_samples.py \
    --input data/verilog_eval.json \
    --output data/test.json \
    --max-samples 1 \
    --verbose
```

### Q: 筛选速度很慢
**A**: 每个样本需要调用一次判断模型，可以：
- 使用 `--max-samples` 限制数量
- 确保vLLM服务性能正常
- 考虑并行处理（需要修改脚本）
