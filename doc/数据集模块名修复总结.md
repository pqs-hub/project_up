# 数据集模块名修复总结

## 问题描述

在运行束搜索攻击引擎时发现所有 Testbench 验证都失败，根本原因是：

**RTL 代码和 Testbench 中的模块名不匹配**

### 典型问题示例

```verilog
// RTL 代码
module RefModule(...)  // ← 模块名是 RefModule
    ...
endmodule

// Testbench
module testbench;
    top_module dut(...);  // ← 实例化的是 top_module，找不到！
endmodule
```

这导致 iverilog 编译失败，所有样本的 TB 验证都无法通过。

---

## 修复方案

统一所有数据集中的模块名为 **`top_module`**：
- RTL 代码：`module top_module`
- Testbench：`top_module dut(...)`

## 技术细节

### 修复逻辑

**RTL 代码修复**:
```python
# 将 module RefModule 替换为 module top_module
rtl_code = re.sub(r'module\s+\w+', 'module top_module', rtl_code, count=1)
```

**Testbench 修复**:
```python
# 将 RefModule dut(...) 替换为 top_module dut(...)
tb_code = re.sub(
    r'(\w+)\s+(dut|uut|inst)\s*\(',
    'top_module \\2(',
    tb_code
)
```
