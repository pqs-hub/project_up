# 快速开始：生成攻击训练数据集

本指南帮助您快速生成高质量的Verilog代码混淆攻击训练数据集。

## 🎯 目标

通过遍历所有攻击规则，对启用CoT的判断模型进行攻击，生成只包含成功样本的高质量训练数据集。

## 📋 前提条件

### 1. 启动服务

```bash
# 终端1: 启动判断模型（CoT模式）
CUDA_VISIBLE_DEVICES=2,3 python -m vllm.entrypoints.openai.api_server \
    --model "/mnt/public/pqs/Model/Qwen2.5-Coder-7B/" \
    --host 0.0.0.0 \
    --port 8001 \
    --tensor-parallel-size 2

# 终端2: 启动攻击模型（用于LLM参数生成，可选）
CUDA_VISIBLE_DEVICES=4,5 python -m vllm.entrypoints.openai.api_server \
    --model "/mnt/public/pqs/Veri_atack/project_up/models" \
    --host 0.0.0.0 \
    --port 8002 \
    --tensor-parallel-size 2
```

### 2. 准备数据

确保有筛选后的评估数据：

```bash
python pipeline/0_filter_correct_samples.py \
    --input data/verilog_eval.json \
    --output data/verilog_eval_correct_only.json \
    --verbose
```

## 🚀 三步生成数据集

### 第一步：生成原始数据集

```bash
# 基础版本（使用默认参数）
python pipeline/6_generate_attack_dataset.py \
    --eval-file data/verilog_eval_correct_only.json \
    --output data/attack_dataset_base.jsonl \
    --max-samples 100 \
    --use-cot \
    --verbose
```

**预期输出**：
- 100个样本 × 8个规则 = 最多800次尝试
- 实际成功样本：约100-200个（取决于规则效果）
- 运行时间：约30-60分钟

### 第二步：分析数据集质量

```bash
python pipeline/7_analyze_attack_dataset.py \
    data/attack_dataset_base.jsonl \
    --stats \
    --quality
```

**示例输出**：
```
============================================================
数据集分析报告
============================================================
总样本数: 150
唯一任务数: 45
平均置信度: 0.8234
Testbench通过率: 100.00%

规则分布:
  T45: 60 (40.0%)
  T19: 35 (23.3%)
  T34: 25 (16.7%)
  T20: 20 (13.3%)
  T07: 10 (6.7%)
============================================================
```

### 第三步：转换为训练格式

```bash
python pipeline/7_analyze_attack_dataset.py \
    data/attack_dataset_base.jsonl \
    --filter-rules T19,T20,T34,T45 \
    --min-confidence 0.7 \
    --to-sft data/sft_training_dataset.jsonl \
    --stats
```

**结果**：
- `sft_training_dataset.jsonl`：可直接用于微调训练

## 🎨 高级用法

### 启用LLM参数生成

使用LLM生成更多样化的攻击参数：

```bash
python pipeline/6_generate_attack_dataset.py \
    --eval-file data/verilog_eval_correct_only.json \
    --output data/attack_dataset_llm.jsonl \
    --max-samples 100 \
    --use-cot \
    --enable-llm-params \
    --attack-base-url http://localhost:8002/v1 \
    --attack-model "obfuscation_merged" \
    --verbose
```

**优势**：
- ✅ T20规则：生成多样化的误导性注释
- ✅ T34规则：生成针对性的对抗重命名
- ✅ T19规则：生成定制化的死代码

**代价**：
- ⚠️ 速度更慢（每个样本多调用几次API）
- ⚠️ 需要攻击模型服务运行

### 只测试特定规则

专注于某些效果好的规则：

```bash
python pipeline/6_generate_attack_dataset.py \
    --eval-file data/verilog_eval_correct_only.json \
    --output data/attack_dataset_focused.jsonl \
    --max-samples 50 \

    --verbose
```

### 分批生成大规模数据集


## 📊 数据集质量指标

### 好的数据集应该满足：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **Testbench通过率** | 100% | 功能等价性 |
| **平均置信度** | ≥ 0.7 | 判断模型被欺骗的置信度 |
| **规则覆盖** | ≥ 4种 | 多样性 |
| **样本数** | ≥ 100 | 训练充分性 |
| **任务覆盖** | ≥ 30个 | 泛化能力 |

## 💡 常见问题

### Q: 为什么成功率不高？
**A**: 成功率取决于：
1. CoT判断模型的强度
2. 攻击规则的有效性
3. 原始代码的复杂度

建议：先用`--max-samples 10`测试，观察哪些规则效果好。

### Q: 可以并行运行吗？
**A**: 可以！将数据集分片，使用不同的`--rules`参数并行运行：

```bash
# 终端1
python pipeline/6_generate_attack_dataset.py --rules T19,T20 ...

# 终端2
python pipeline/6_generate_attack_dataset.py --rules T34,T45 ...

# 终端3
python pipeline/6_generate_attack_dataset.py --rules T07,T47 ...
```

### Q: 内存不足怎么办？
**A**: 
1. 减少`--max-samples`
2. 分批运行
3. 使用`--rules`限制规则数量
