Design a simple PCIe interface state machine that manages the state transitions between IDLE, REQUEST, and RESPONSE states. The state machine should have the following behavior:
- It starts in the IDLE state.
- On receiving a 'start' signal, it transitions to the REQUEST state.
- In the REQUEST state, if a 'ack' signal is received, it transitions to the RESPONSE state.
- In the RESPONSE state, after a fixed number of clock cycles (determined by a parameter), it transitions back to the IDLE state.
The state machine should also have an output that indicates the current state.
