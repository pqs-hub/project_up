## 📈 详细对比

### 1. 原始版本（sft_attack_success_registry.jsonl）

```
文件: data/sft_attack_success_registry.jsonl
样本数: 44,246
独特任务: 16,126
平均每任务规则数: 2.74
```

#### 规则分布
```
T19 (ASR=90.8%): 14,942 (33.8%)
T20 (ASR=51.6%):  8,519 (19.3%)
T34 (ASR=42.1%):  6,679 (15.1%)
T45 (ASR=99.1%):  6,199 (14.0%)
其他规则:         8,907 (17.8%)
```

#### 问题
⚠️ **93.7%样本存在冲突**
- 同一input对应多个output
- 训练时梯度冲突
- 模型预测不稳定

#### 适用场景
❌ **不推荐直接使用**（除非修改训练方式）

---

### 2. 完全去重版本（sft_attack_success_dedup.jsonl）

```
文件: data/sft_attack_success_dedup.jsonl
样本数: 16,126
独特任务: 16,126
平均每任务规则数: 1.0
```

#### 规则分布
```
T45 (ASR=99.1%): 6,193 (38.4%) ← 占比最大
T19 (ASR=90.8%): 9,218 (57.2%) ← 占比最大
T20 (ASR=51.6%):   490 ( 3.0%)
T34 (ASR=42.1%):   175 ( 1.1%)
其他规则:           50 ( 0.4%) ← 样本极少
```

#### 优点
✅ **训练稳定** - 无冲突
✅ **聚焦高效** - 保留ASR最高的规则
✅ **实施简单** - 直接可用

#### 缺点
⚠️ **规则极不平衡** - T45+T19占95.6%
⚠️ **规则多样性差** - 低ASR规则样本<0.5%
⚠️ **泛化能力弱** - 模型可能只学会2个规则

#### 适用场景
✅ 快速验证可行性
✅ 只关注高ASR规则
❌ 需要规则多样性时不推荐

---

### 3. 平衡去重版本（sft_attack_success_balanced.jsonl）⭐ 推荐

```
文件: data/sft_attack_success_balanced.jsonl
样本数: 37,745
独特任务: 16,126
平均每任务规则数: 2.34

```


**📊 中ASR规则 (40% - 80%)**: 13,630 (36.1%)
```

T12 (ASR=56.5%):   560 ( 1.5%)

```

**📉 低ASR规则 (< 40%)**: 5,182 (13.7%)
```
T03 (ASR=16.7%): 1,924 ( 5.1%)
T31 (ASR=36.8%):   932 ( 2.5%)
T32 (ASR=14.4%):   926 ( 2.5%)
T41 (ASR=34.2%):   691 ( 1.8%)
T48 (ASR=28.4%):   284 ( 0.8%)
T09 (ASR=37.1%):   249 ( 0.7%)
T07 (ASR=22.4%):   106 ( 0.3%)
T10 (ASR=26.2%):    63 ( 0.2%)
T47 (ASR=28.2%):     7 ( 0.0%)
```

#### 优点
✅ **规则全覆盖** - 15/15规则都有样本
✅ **分布均衡** - 高/中/低ASR规则都有足够样本
✅ **数据量大** - 保留85.3%样本
✅ **训练相对稳定** - 每任务最多3个规则（冲突降低70%）
✅ **泛化能力强** - 模型能学到多种混淆策略

#### 缺点
⚠️ **仍有少量冲突** - ~30%样本有冲突（但可接受）
⚠️ **实施稍复杂** - 需要分层采样逻辑

#### 适用场景
✅ **推荐作为主数据集** - 平衡效果和多样性
✅ 需要规则多样性
✅ 希望模型泛化能力强
✅ 可接受轻微冲突（训练时shuffle可缓解）

## 🎓 经验总结

### 关键发现
1. ✅ **多规则冲突确实是严重问题**（82.6%任务被多规则攻击）
2. ✅ **完全去重解决冲突但损失多样性**（T45+T19占95.6%）
3. ✅ **分层采样是最佳平衡**（保留多样性+降低冲突）

### 最佳实践
1. **数据集选择**: 用平衡去重版本（37,745样本）
2. **训练策略**: 使用shuffle缓解剩余冲突
3. **评估指标**: 关注规则多样性和功能等价率
4. **迭代优化**: 根据评估结果调整采样策略

### 未来改进
1. **自适应采样**: 根据规则难度动态调整采样比例
2. **多目标优化**: 同时优化ASR和规则多样性
3. **课程学习**: 先学高ASR规则，再学低ASR规则
